    ```typescript
    import type * as ExcelJS from 'exceljs';
    import type { ImportFailureDto } from '../dto';

    export interface ColumnDefinition {
      header: string;
      key: string;
      width: number;
    }

    export interface ImportStrategy {
      /** Column definitions for template generation */
      getColumns(): ColumnDefinition[];

      /** Get instructions for template instructions sheet */
      getInstructions(): Array<{ field: string; required: string; description: string }>;

      /** Check if this strategy matches Excel headers */
      matchesHeaders(headers: string[]): boolean;

      /** Get existing entity keys for duplicate detection */
      getExistingKeys(): Promise<Set<string>>;

      /** Validate a single row, return failure or null if valid, plus skipped flag */
      validateRow(
        row: ExcelJS.Row,
        rowNumber: number,
        batchKeys: Set<string>,
        existingKeys: Set<string>,
      ): { valid: boolean; failure?: ImportFailureDto; skipped?: boolean };

      /** Transform validated row to entity creation data */
      transformRow(row: ExcelJS.Row): Record<string, unknown>;

      /** Bulk create entities in database */
      createBatch(entities: Record<string, unknown>[]): Promise<number>;
    }
    ```

    **Step 1: Refactor ImportService** (`backend/src/import/import.service.ts`)

    Transform from monolithic to orchestrator. The new ImportService should be approximately 150-200 lines (down from 607).

    Constructor now injects strategies directly:
    ```typescript
    constructor(
      private readonly prisma: PrismaService,
      private readonly fabricStrategy: FabricImportStrategy,
      private readonly supplierStrategy: SupplierImportStrategy,
    ) {}
    ```

    New private method `detectStrategy(worksheet: ExcelJS.Worksheet): ImportStrategy`:
    - Read row 1 headers via `worksheet.getRow(1)`
    - Extract header values: iterate cells, collect string values into array
    - Check `this.fabricStrategy.matchesHeaders(headers)` first
    - Then check `this.supplierStrategy.matchesHeaders(headers)`
    - If neither matches, throw `BadRequestException('Unable to detect import type from column headers')`

    Refactor `importFabrics(buffer: Buffer)` and `importSuppliers(buffer: Buffer)` into a single generic method `importData(buffer: Buffer): Promise<ImportResultDto>`:
    - Load workbook from buffer (using ExcelJS)
    - Get first worksheet
    - Call `detectStrategy(worksheet)` to get the appropriate strategy
    - Call `strategy.getExistingKeys()` for duplicate detection
    - Iterate rows (skip header row), for each row:
      - Call `strategy.validateRow(row, rowNumber, batchKeys, existingKeys)`
      - If skipped, increment skipped count
      - If invalid, add failure to failures array, increment failed count
      - If valid, call `strategy.transformRow(row)` and add to batch
    - Call `strategy.createBatch(batch)` for all valid rows
    - Return `ImportResultDto` with counts and failures

    Keep the old `importFabrics()` and `importSuppliers()` methods as thin wrappers that call `importData()` — this preserves the controller API:
    ```typescript
    async importFabrics(buffer: Buffer): Promise<ImportResultDto> {
      return this.importData(buffer);
    }
    async importSuppliers(buffer: Buffer): Promise<ImportResultDto> {
      return this.importData(buffer);
    }
    ```
    Note: since `importData` auto-detects strategy from headers, both methods can delegate to it. The controller still calls the correct method name.
