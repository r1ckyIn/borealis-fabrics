Modify `.github/workflows/ci.yml` to add a `security` job and pnpm store caching on existing jobs.

**Change 1: Fix pnpm/setup-node ordering in existing `backend` job**

The current ci.yml has `actions/setup-node` BEFORE `pnpm/action-setup`. For caching to work, pnpm must be installed FIRST so setup-node can find the store path. Reorder steps and add cache config:

```yaml
backend:
  name: Backend
  runs-on: ubuntu-latest
  defaults:
    run:
      working-directory: backend
  steps:
    - uses: actions/checkout@v4

    - name: Install pnpm
      uses: pnpm/action-setup@v4
      with:
        version: 10

    - name: Setup Node.js
      uses: actions/setup-node@v4
      with:
        node-version: 22
        cache: 'pnpm'
        cache-dependency-path: backend/pnpm-lock.yaml

    - name: Install dependencies
      run: pnpm install --frozen-lockfile

    - name: Generate Prisma Client
      run: npx prisma generate

    - name: Lint
      run: pnpm lint

    - name: Type check
      run: npx tsc --noEmit

    - name: Unit tests
      run: pnpm exec jest --passWithNoTests

    - name: Build
      run: pnpm build
```

Key changes to backend job:
- Move `pnpm/action-setup` BEFORE `actions/setup-node` (required for cache detection)
- Add `cache: 'pnpm'` and `cache-dependency-path: backend/pnpm-lock.yaml` to setup-node

**Change 3: Add new `security` job**

Add this as a third job, running in parallel with backend and frontend (no `needs` dependency):

```yaml
security:
  name: Security Audit
  runs-on: ubuntu-latest
  # Non-blocking initially; graduate to required once allowlist is stable
  continue-on-error: true
  steps:
    - uses: actions/checkout@v4

    - name: Install pnpm
      uses: pnpm/action-setup@v4
      with:
        version: 10

    - name: Setup Node.js
      uses: actions/setup-node@v4
      with:
        node-version: 22

    - name: Install backend dependencies
      working-directory: backend
      run: pnpm install --frozen-lockfile

    - name: Audit backend
      working-directory: backend
      run: pnpm dlx audit-ci@^7 --config ../audit-ci.jsonc

    - name: Install frontend dependencies
      working-directory: frontend
      run: pnpm install --frozen-lockfile

    - name: Audit frontend
      working-directory: frontend
      run: pnpm dlx audit-ci@^7 --config ../audit-ci.jsonc
```

Key design decisions:
- `continue-on-error: true` -- non-blocking initially to avoid breaking existing PRs due to newly disclosed CVEs. Graduate to required check once allowlist is stable.
- No `cache` on security job setup-node -- the security job does NOT need caching since it runs `pnpm dlx` (which uses a temporary store). Adding cache would just waste cache space.
- `pnpm dlx audit-ci@^7` -- uses pnpm's dlx (equivalent to npx) to run audit-ci without installing it as a project dependency. The `@^7` ensures latest v7.x.
- `--config ../audit-ci.jsonc` -- relative path from backend/ or frontend/ working directory to the project-root config file.
- No `--high` flag needed -- the `"high": true` in audit-ci.jsonc already sets the threshold.

The final ci.yml should have exactly 3 jobs: `backend`, `frontend`, `security`. The `on` triggers remain unchanged (push to main/develop, PR to main).
