# Plan 13-06: Fix includeDeleted RBAC â€” OptionalJwtAuthGuard

## Problem

The `findAll` endpoints on 4 entity controllers (supplier, customer, fabric, product) lack `@UseGuards(JwtAuthGuard)`. When a user sends `includeDeleted=true`, the RBAC check in the controller reads `this.cls.get('user')` which returns `undefined` because no guard populated `request.user`. This forces `includeDeleted=false` regardless of the user's admin status.

The `$raw` PrismaClient bypass (13-05) works correctly â€” verified independently returning 43 records including 2 deleted. The bug is purely in the auth guard layer.

## Root Cause

```
Request â†’ (no guard) â†’ UserClsInterceptor (request.user=undefined â†’ CLS user=undefined) â†’ Controller RBAC (!user â†’ includeDeleted=false) â†’ Service (never uses $raw)
```

## Solution

Create `OptionalJwtAuthGuard` that:
1. Extracts JWT from cookie/header if present
2. Validates and sets `request.user` if valid
3. Returns `true` (allows request) even if no token or invalid token
4. Does NOT throw 401 for unauthenticated requests

Apply to all 4 `findAll` endpoints so the RBAC check can identify admin users.
