# Plan 13-06: Fix includeDeleted RBAC + CI Failures

## Problem 1: includeDeleted RBAC

The `findAll` endpoints on 4 entity controllers lack auth guards. When `includeDeleted=true` is sent, the RBAC check reads `this.cls.get('user')` → `undefined` → forces `includeDeleted=false`.

## Problem 2: CI TypeScript Failures (15 consecutive failures)

Three categories of CI-only TypeScript errors (local build passes, CI strict check fails):

1. **Buffer type mismatch** (7 errors in `export.service.spec.ts`):
   `Buffer.from(arrayBuffer)` returns `Buffer<ArrayBufferLike>` which is not assignable to `Buffer` in Node 22 strict typing. `ExcelJS.Workbook.xlsx.load()` expects `Buffer`.

2. **Stale mock** (1 error in `system.controller.spec.ts`):
   `mockEnumsResponse` missing `productCategory` and `productSubCategory` fields added to `EnumsResponseDto`.

3. **Constructor signature mismatch** (10 errors across all E2E specs):
   `new AllExceptionsFilter()` called with 0 args, but constructor requires `ClsService` (added in Phase 12).

## Tasks

### Task 1: Create OptionalJwtAuthGuard

Create `backend/src/auth/guards/optional-jwt-auth.guard.ts`:
- Compose with existing `JwtAuthGuard` logic
- Try extracting and validating token; if valid, set `request.user`
- On missing/invalid token: silently pass (return `true`, don't throw)
- Export from guards index

**AC:**
- Guard exists and is exported
- Valid token → request.user populated
- Missing/invalid token → request through, request.user undefined
- Build passes

### Task 2: Apply OptionalJwtAuthGuard to findAll endpoints

Add `@UseGuards(OptionalJwtAuthGuard)` to `findAll` on:
- `supplier.controller.ts`
- `customer.controller.ts`
- `fabric.controller.ts`
- `product.controller.ts`

**AC:**
- All 4 controllers have the guard on findAll
- With valid admin token + includeDeleted=true → returns deleted records
- Without token + includeDeleted=true → includeDeleted forced to false
- Build + test + lint pass

### Task 3: Fix Buffer type and stale mock in unit tests

**3a. Fix `export.service.spec.ts` Buffer type (7 errors):**
Cast `Buffer.from(arrayBuffer)` result to `Buffer` in the service, or cast in the test's `workbook.xlsx.load()` call. The simplest fix: cast to `Buffer` in the service return type.

**3b. Fix `system.controller.spec.ts` stale mock (1 error):**
Add `productCategory` and `productSubCategory` to `mockEnumsResponse` to match current `EnumsResponseDto`.

**AC:**
- `npx tsc --noEmit` passes with 0 errors
- `pnpm test` passes

### Task 4: Fix AllExceptionsFilter constructor in E2E tests

All 10 E2E spec files call `new AllExceptionsFilter()` with no arguments. Since Phase 12 added `ClsService` to the constructor, all E2E files need updating.

Fix: In each E2E `beforeAll()`, get `ClsService` from the testing module and pass it:
```typescript
const cls = app.get(ClsService);
app.useGlobalFilters(new AllExceptionsFilter(cls));
```
