### Task 13-05-02: Fix includeDeleted Soft-Delete Bypass

<read_first>
- backend/src/prisma/prisma.service.ts (full file — current extension setup)
- backend/src/supplier/supplier.service.ts (lines 70-100 — current includeDeleted logic)
- backend/src/customer/customer.service.ts (includeDeleted logic)
- backend/src/fabric/fabric.service.ts (includeDeleted logic)
- backend/src/product/product.service.ts (includeDeleted logic)
- backend/node_modules/prisma-extension-soft-delete/dist/lib/helpers/createParams.js (extension source — line 244: `where[config.field] || config.createValue(false)`)
- backend/src/supplier/supplier.service.spec.ts (existing tests)
</read_first>

<action>
**Root Cause**: The `where.deletedAt = {} as Prisma.DateTimeNullableFilter` bypass doesn't work because Prisma's query engine treats an empty object `{}` as a no-op filter rather than a "match all" filter. The soft-delete extension sees a truthy value and doesn't override it, but Prisma itself generates no SQL condition from `{}`, effectively still filtering by `deletedAt IS NULL` (default behavior).

**Fix Strategy**: Expose a raw (unextended) PrismaClient on PrismaService. Admin queries that need includeDeleted use this raw client to completely bypass the soft-delete extension.

**Step 1: Modify `backend/src/prisma/prisma.service.ts`**

Add a raw PrismaClient property that skips the soft-delete extension:

```typescript
@Injectable()
export class PrismaService extends PrismaClient implements OnModuleInit, OnModuleDestroy {
  /**
   * Unfiltered PrismaClient — bypasses soft-delete extension.
   * Use for admin queries that need to include soft-deleted records.
   */
  readonly $raw: PrismaClient;

  constructor() {
    super();
    this.$raw = new PrismaClient();
    // ... existing extension code (no changes)
  }

  async onModuleInit() {
    await this.$connect();
    await this.$raw.$connect();
  }

  async onModuleDestroy() {
    await this.$raw.$disconnect();
    await this.$disconnect();
  }
}
```

**Step 2: Update 4 entity services to use `this.prisma.$raw` when `includeDeleted=true`**

In each of the 4 services (supplier, customer, fabric, product), replace the broken bypass pattern:

```typescript
// BEFORE (broken):
if (query.includeDeleted) {
  where.deletedAt = {} as Prisma.DateTimeNullableFilter;
}
const [data, total] = await Promise.all([
  this.prisma.supplier.findMany({ where, skip, take, orderBy }),
  this.prisma.supplier.count({ where }),
]);

// AFTER (working):
const client = query.includeDeleted ? this.prisma.$raw : this.prisma;
const [data, total] = await Promise.all([
  client.supplier.findMany({ where, skip, take, orderBy }),
  client.supplier.count({ where }),
]);
```

Remove the `where.deletedAt = {} as Prisma.DateTimeNullableFilter` line from all 4 services.

The 4 files and approximate line locations:
- `backend/src/supplier/supplier.service.ts` — lines 77-80, change findMany/count around line 110+
- `backend/src/customer/customer.service.ts` — lines 133-135, change findMany/count
