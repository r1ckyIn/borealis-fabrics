  <action>
    Create standalone migration script (`backend/src/scripts/migrate-to-cos.ts`):

    ```typescript
    /**
     * One-time migration script: FabricImage.url localhost URLs -> key-only format.
     *
     * What it does:
     * 1. Finds all FabricImage records with URLs starting with 'http://localhost'
     * 2. Extracts the key from the URL (everything after '/uploads/')
     * 3. If COS mode: reads local file and uploads to COS
     * 4. Updates database record to store key-only
     *
     * Usage:
     *   DRY_RUN=true npx ts-node src/scripts/migrate-to-cos.ts    # Preview changes
     *   npx ts-node src/scripts/migrate-to-cos.ts                  # Execute migration
     *   STORAGE_MODE=cos npx ts-node src/scripts/migrate-to-cos.ts # Upload to COS + update DB
     *
     * Prerequisites:
     * - Deploy new code (that reads key-only) BEFORE running this script
     * - Set environment variables (.env) for database and optionally COS
     */
    ```

    Implementation details:
    - Import: `PrismaClient` from `@prisma/client`, `fs`, `path`, `dotenv`
    - Load env: `dotenv.config()`
    - Create PrismaClient instance
    - Read env vars: `DRY_RUN = process.env.DRY_RUN === 'true'`, `STORAGE_MODE = process.env.STORAGE_MODE || 'local'`, `UPLOAD_DIR = process.env.UPLOAD_DIR || './uploads'`

    - If STORAGE_MODE is 'cos', create COS client:
      ```typescript
      import COS from 'cos-nodejs-sdk-v5';
      const cos = new COS({
        SecretId: process.env.COS_SECRET_ID!,
        SecretKey: process.env.COS_SECRET_KEY!,
      });
      const bucket = process.env.COS_BUCKET!;
      const region = process.env.COS_REGION!;
      ```

    - Main function `migrateFabricImagesToCos`:
      a. Query: `prisma.fabricImage.findMany({ where: { url: { startsWith: 'http' } } })`
      b. Log: `Found ${images.length} images with full URLs to migrate`
      c. If `images.length === 0`, log 'No images to migrate' and exit
      d. Track stats: `let migrated = 0, skipped = 0, failed = 0`
      e. For each image:
         - Extract key: `const key = image.url.split('/uploads/').pop()` — if no '/uploads/' found, log warning, increment skipped, continue
         - If DRY_RUN: log `[DRY RUN] Would migrate image ${image.id}: ${image.url} -> ${key}`, increment migrated, continue
         - If STORAGE_MODE is 'cos':
           - Read local file: `const localPath = path.resolve(UPLOAD_DIR, key)`
           - If file doesn't exist: log warning, increment skipped, continue
           - Read buffer: `const buffer = await fs.promises.readFile(localPath)`
           - Determine mime type from extension (simple map: .jpg/.jpeg -> image/jpeg, .png -> image/png, .gif -> image/gif, .webp -> image/webp, default -> application/octet-stream)
           - Upload to COS via Promise wrapper:
             ```typescript
             await new Promise<void>((resolve, reject) => {
               cos.putObject({ Bucket: bucket, Region: region, Key: key, Body: buffer, ContentType: mimeType }, (err) => err ? reject(err) : resolve());
             });
             ```
         - Update database: `await prisma.fabricImage.update({ where: { id: image.id }, data: { url: key } })`
         - Log: `Migrated image ${image.id}: ${image.url} -> ${key}`
         - Increment migrated
         - On error: log error, increment failed, continue (don't abort entire migration)

      f. Log summary: `Migration complete: ${migrated} migrated, ${skipped} skipped, ${failed} failed out of ${images.length} total`

    - Execute: call `migrateFabricImagesToCos().catch(console.error).finally(() => prisma.$disconnect())`
