Replace the current List-based layout in CustomerAddressTab.tsx with a responsive card grid using Ant Design Row/Col/Card components. The props interface (CustomerAddressTabProps) MUST remain unchanged.

**Import changes:**
- Remove: `List` from antd imports
- Add: `Row, Col, Card, Typography` from antd imports
- Keep: `Empty, Tag, Space` from antd imports
- Keep: `EnvironmentOutlined, StarFilled` from '@ant-design/icons'

**Layout structure:**
Replace the `<List>` with `<Row gutter={[16, 16]}>` containing `<Col>` items wrapping `<Card>` components:

```tsx
<Row gutter={[16, 16]}>
  {addresses.map((address, index) => (
    <Col xs={24} md={12} key={index}>
      <Card size="small" hoverable>
        <Space direction="vertical" size={4} style={{ width: '100%' }}>
          <Space>
            <EnvironmentOutlined className="text-blue-500" />
            <Text strong>{address.contactName}</Text>
            <Text type="secondary">{address.contactPhone}</Text>
            {address.label && <Tag color="blue">{address.label}</Tag>}
            {address.isDefault && (
              <Tag color="gold" icon={<StarFilled />}>默认</Tag>
            )}
          </Space>
          <Text type="secondary" style={{ paddingLeft: 22 }}>
            {formatAddress(address)}
          </Text>
        </Space>
      </Card>
    </Col>
  ))}
</Row>
```

**Key design decisions:**
- `xs={24} md={12}`: 1 column on mobile, 2 columns on desktop (addresses are typically few, 2 cols is more compact)
- `Card size="small"`: Reduces internal Card padding for a denser look
- `hoverable`: Subtle interactivity hint
- Contact name + phone on same line as icon and tags (horizontal, not vertical via List.Item.Meta)
- Full address text below with `paddingLeft: 22` to align under the text (past the icon)
- `gutter={[16, 16]}`: 16px horizontal and vertical spacing between cards

**Empty state:** Keep the existing `if (!addresses || addresses.length === 0)` guard returning `<Empty description="暂无收货地址" />` unchanged.

**formatAddress function:** Keep unchanged.

**Do NOT:**
- Change the exported interface `CustomerAddressTabProps`
- Change the function name or export pattern
- Add any new props or state
