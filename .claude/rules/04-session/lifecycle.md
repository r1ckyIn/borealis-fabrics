# Session 生命周期管理

本文件定义了 Claude Code session 的生命周期管理规范。

---

## Session 开始

```text
Claude 必须在 session 开始时：
1. 确认已读取相关 CLAUDE.md（自动完成）
2. 如果是继续之前的任务，确认上下文
3. 如果是新任务，阅读需求文档（如存在）
```

---

## Session 进行中

```text
Claude 必须在 session 中：
1. 发现新模式时用 💡 标记
2. 遇到需要记住的配置陷阱时记录
3. 长 session（>50 轮对话）时建议 /clear
```

---

## Session 结束前

```text
用户说"结束"、"done"、"完成"或准备 /clear 时：
1. 回顾 session 中标记的 💡 发现
2. 提议需要记录的模式
3. 确认用户是否要更新 CLAUDE.md
4. 执行自我迭代检查清单
```

---

## /clear 前检查（强制）

```text
用户执行 /clear 前，Claude 必须：
□ 确认没有未完成的任务
□ 确认没有未记录的重要发现
□ 如果有发现，先提议更新再 /clear
```

---

## 自我迭代检查清单

```text
自我迭代检查清单（每次任务后在心中执行）：

□ 我是否遵循了安全 AI 辅助开发工作流？
□ 当用户试图跳过步骤时我是否提醒了？
□ 我是否在提交前运行了 build/test/lint？
□ 我是否为 PR 建议了 /code-review？
□ 是否有我应该捕获的安全警告？
□ 我是否发现了需要添加到 CLAUDE.md 的新模式？

如果发现任何问题：
1. 记录出了什么问题
2. 建议用新规则更新 CLAUDE.md
3. 在下次交互中应用所学

迭代日志（重要时记录）：
"自我迭代：发现 [问题]，建议向 CLAUDE.md 添加 [新规则]"
```

---

## Session 类型

### 新任务 Session

```text
开始新任务时：
1. 阅读 CLAUDE.md 和需求文档
2. 探索代码库了解结构
3. 制定计划并获得批准
4. 按计划执行
```

### 继续任务 Session

```text
继续之前的任务时：
1. 确认上下文和进度
2. 回顾之前的工作
3. 继续执行
```

### 调试 Session

```text
调试问题时：
1. 理解问题描述
2. 复现问题
3. 定位根因
4. 修复并验证
5. 记录解决方案
```

---

## Session 管理最佳实践

### 一个 Session = 一个模块的一个功能

保持 session 聚焦，避免上下文混乱。

### 长 Session 管理

- 对话超过 50 轮时建议 /clear
- 完成一个完整功能模块后建议 /clear
- 切换到完全不同的任务时建议 /clear

### 进度同步

- 每完成一步验证就更新文档
- session 之间用 CLAUDE.md 传递关键信息
