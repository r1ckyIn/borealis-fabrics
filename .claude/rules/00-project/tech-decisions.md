# 关键技术决策

本文件记录项目中的重要技术决策。

---

## 已确定决策

| 决策 | 结论 | 理由 |
|------|------|------|
| 付款与订单状态 | MVP 不绑定，仅提醒 | 简化 MVP 复杂度 |
| Excel 导入冲突 | 跳过已存在，不覆盖 | 避免数据丢失 |
| 编号序号位数 | 4 位（BF-YYMM-NNNN） | 满足日常业务量 |
| 客户地址存储 | JSON + AddressVO 封装 | 灵活支持多地址 |
| isActive vs status | isActive=软删除，status=业务状态 | 职责分离 |
| 前端测试 | Vitest | 与 Vite 集成好 |
| 后端测试 | Jest + SuperTest | NestJS 官方推荐 |

---

## 需求确认规则（强制）

在编写或修改需求文档时，对以下假设必须主动与用户确认，不可直接沿用文档中的表述：

- 目标平台和使用场景（桌面/移动/混合，主要使用比例）
- 用户角色和实际工作环境
- 核心业务流程的真实操作方式
- 技术选型中与用户体验直接相关的决策（如 UI 组件库）

## 新增技术决策模板

当需要记录新的技术决策时，使用以下格式：

```markdown
### [YYYY-MM-DD] 决策标题

**背景**：为什么需要做这个决定

**选项**：
1. 选项 A - 优点/缺点
2. 选项 B - 优点/缺点

**决定**：选择哪个选项

**理由**：为什么选择这个

**影响**：这个决定的影响范围
```
