# 生产级质量检查清单

本文件定义了生产级项目的质量检查标准。

---

## 基础质量检查清单

推送到 GitHub 前：

- [ ] **本地构建通过**
- [ ] **本地测试通过**
- [ ] **本地 lint 通过**
- [ ] README.md 为双语（EN + CN）
- [ ] 所有徽章使用 `flat-square` 风格
- [ ] LICENSE 文件存在（MIT）
- [ ] .gitignore 适合相应语言
- [ ] .gitignore 包含 CLAUDE.md
- [ ] 代码中无密钥/凭据
- [ ] 代码遵循风格指南
- [ ] 代码注释为纯英文
- [ ] 项目结构匹配模板
- [ ] 类型注解完整
- [ ] 逐功能验证已完成
- [ ] 模块集成已测试
- [ ] 全系统验证已通过

---

## 生产级项目额外检查清单

### 架构

- [ ] Clean Architecture 依赖方向正确（单向）
- [ ] 前端 Feature-Based 模块化
- [ ] 关注点分离

### 数据库

- [ ] 数据库迁移文件已生成并提交
- [ ] 迁移可回滚
- [ ] 索引合理

### 配置

- [ ] 环境变量通过 .env.example 记录（无真实值）
- [ ] .env 文件在 .gitignore 中
- [ ] pydantic-settings 验证配置

### CI/CD

- [ ] Pre-commit hooks 配置正确
- [ ] CI/CD Pipeline 配置完整（lint → type check → test → build）
- [ ] 自动部署到 staging
- [ ] 生产部署需要审批

### 测试

- [ ] 测试覆盖率 >= 80%（核心 domain >= 95%）
- [ ] 单元测试覆盖核心业务逻辑
- [ ] 集成测试覆盖 API 端点
- [ ] E2E 测试覆盖关键路径

### 文档

- [ ] ADR 记录了重要架构决策
- [ ] API 文档完整（OpenAPI）
- [ ] 数据库设计文档

### 容器化

- [ ] Docker/docker-compose 本地环境可正常启动
- [ ] Dockerfile 优化（多阶段构建）
- [ ] 容器安全扫描

### 监控

- [ ] 健康检查端点可用（/health, /ready）
- [ ] Sentry 错误追踪已配置
- [ ] 结构化日志配置

### 类型安全

- [ ] TypeScript strict 模式，无 any
- [ ] Python mypy strict 模式
- [ ] 所有 API 端点有 Pydantic schema 验证

### 安全

- [ ] 安全依赖扫描通过（safety / snyk）
- [ ] 无硬编码密钥
- [ ] 输入验证完整
- [ ] 错误消息不泄露敏感信息

## 上线前终极检查

### 功能验证

- [ ] 所有功能按需求工作
- [ ] 边界情况已处理
- [ ] 错误处理完整

### 性能验证

- [ ] API 响应时间 < 500ms (P95)
- [ ] 无 N+1 查询问题
- [ ] 资源正确释放

### 安全验证

- [ ] 认证/授权正确
- [ ] 敏感数据加密
- [ ] CORS 配置正确

### 运维准备

- [ ] 监控告警配置
- [ ] 日志可搜索
- [ ] 备份策略就绪
- [ ] 回滚计划准备

---

## 检查清单使用方法

### 每日开发

1. 功能完成后运行基础检查
2. 提交前验证 lint/test/build

### PR 创建

1. 运行完整基础检查清单
2. 运行 `/code-review`

### 上线部署

1. 运行完整生产级检查清单
2. 运行上线前终极检查
3. 在 staging 验证
4. 审批后部署生产
