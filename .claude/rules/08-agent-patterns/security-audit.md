# 安全审查模式

本文件定义了安全审查的标准 agent 调用模式。

---

## 触发条件

- PR 前安全检查
- 代码审查中的安全部分
- 定期安全审计

---

## 调用方式

使用 `Task(Explore)` 或 `/code-review`

---

## Prompt 模板

```text
对以下文件进行安全审查：
<file list>

检查项：
□ 无硬编码密钥
  - 搜索 "password", "secret", "key", "token"
  - 检查配置文件
  - 检查环境变量使用

□ 无 SQL/命令注入
  - 检查 SQL 查询是否参数化
  - 检查 shell 命令是否安全
  - 检查用户输入处理

□ 输入验证完整
  - 检查所有外部输入
  - 检查文件上传
  - 检查 URL 参数

□ 错误消息不泄露敏感信息
  - 检查错误响应
  - 检查日志输出
  - 检查调试信息

报告格式：
1. 问题描述
2. 文件位置
3. 风险等级（高/中/低）
4. 修复建议
```

## 安全检查清单

### 认证与授权

- [ ] 密码正确哈希（bcrypt/argon2）
- [ ] JWT 正确验证
- [ ] 权限检查完整
- [ ] Session 安全配置

### 输入验证

- [ ] 所有用户输入验证
- [ ] 文件上传限制
- [ ] URL 参数验证
- [ ] JSON 请求体验证

### 数据安全

- [ ] 敏感数据加密存储
- [ ] HTTPS 传输
- [ ] 日志不包含敏感数据
- [ ] 错误消息安全

### 配置安全

- [ ] 密钥使用环境变量
- [ ] 调试模式关闭（生产）
- [ ] CORS 正确配置
- [ ] 安全头设置

## 常见漏洞模式

### SQL 注入

问题：字符串拼接 SQL 查询
解决：使用 ORM 参数化查询（如 Prisma）

### 命令注入

问题：直接拼接用户输入到 shell 命令
解决：使用 execFile 替代 exec，或使用项目提供的 execFileNoThrow 工具

### XSS（跨站脚本）

问题：直接将用户输入插入 DOM
解决：使用 textContent 替代 innerHTML，或使用 DOMPurify 进行清理

### 硬编码密钥

问题：密钥写在代码中
解决：使用环境变量（process.env.API_KEY）

## 风险等级定义

| 等级 | 定义 | 示例 |
|------|------|------|
| 高 | 可直接导致数据泄露或系统入侵 | SQL 注入、RCE |
| 中 | 需要特定条件才能利用 | CSRF、敏感信息泄露 |
| 低 | 影响有限 | 信息披露、配置问题 |
