# Session 管理规则

## Session 开始时（强制）

1. 阅读 CLAUDE.md 确认当前进度
2. 确认"当前功能状态"表格是最新的
3. 确定本次 session 要完成的功能

---

## Session 进行中

- 一个 session = 一个模块的一个功能
- 每完成一步验证就更新文档
- 长 session：每完成一个子功能后 /clear

---

## Session 结束时（强制）

1. 更新"当前功能状态"表格
2. 更新"当前开发阶段"的功能进度
3. 如果模块完成，更新 VIBE_CODER_GUIDE
4. 记录任何新的技术决策

---

## 进度跟踪格式

### 功能状态表格

| 功能 | 状态 | 测试 | Build | Lint | 备注 |
|------|------|------|-------|------|------|
| X.X.X 功能名称 | ⏳/🔄/✅ | ⏳/✅ | ⏳/✅ | ⏳/✅ | 备注 |

### 状态图例

- ⏳ 待开始
- 🔄 进行中
- ✅ 已完成
- ❌ 失败待修复

---

## 文档同步检查点（强制）

| 时机 | 必须更新的文件 | 更新内容 |
|------|---------------|---------|
| 功能测试通过后 | CLAUDE.md | 该功能的测试列 → ✅ |
| 功能 Build 通过后 | CLAUDE.md | 该功能的 Build 列 → ✅ |
| 功能 Lint 通过后 | CLAUDE.md | 该功能的 Lint 列 → ✅ |
| 功能提交后 | CLAUDE.md | 该功能状态列 → ✅ 已完成 |
| 模块完成后 | VIBE_CODER_GUIDE | 该模块状态 → ✅ |
| 发现新技术决策 | CLAUDE.md | 关键技术决策表格新增行 |
| Session 结束 | 两个文件 | 当前进度和下一步 |
